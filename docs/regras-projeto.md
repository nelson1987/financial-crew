# Regras do Projeto

Este documento contém todas as regras e convenções alinhadas para o projeto **financial-crew** e pode ser reutilizado como referência em outros projetos.

## Estrutura de Documentação

### Localização
- **Todos os documentos** do projeto devem ser criados na pasta `/docs`
- A documentação funciona como uma **wiki** com referência no README na pasta raiz
- Estrutura similar ao **MkDocs**

### Organização
- Documentos principais devem estar na raiz de `/docs`
- Documentos auxiliares podem ser organizados em subpastas quando necessário
- Sempre manter um `index.md` como ponto de entrada da documentação

## Convenções de Documentação

### Formato
- Todos os documentos devem ser em formato **Markdown (.md)**
- Usar formatação consistente e clara
- Incluir índices e navegação quando apropriado

### Tipos de Documentos

1. **Regras do Projeto** (`regras-projeto.md`)
   - Contém todas as regras, convenções e padrões do projeto
   - Deve ser atualizado sempre que novas regras forem estabelecidas
   - Pode ser reutilizado como template para outros projetos

2. **Tutorial de Comandos** (`comandos-tutorial.md`)
   - Documenta todos os comandos executados durante o desenvolvimento
   - Explica o intuito e propósito de cada comando
   - Serve como referência para reprodução e aprendizado

3. **Índice da Wiki** (`index.md`)
   - Ponto de entrada principal da documentação
   - Contém links para todos os documentos relevantes
   - Organiza a navegação da documentação

## Boas Práticas

### Manutenção
- Sempre atualizar a documentação quando houver mudanças significativas
- Manter os documentos sincronizados com o estado atual do projeto
- Revisar periodicamente para garantir precisão

### Reutilização
- Este documento pode ser copiado e adaptado para outros projetos
- Manter um histórico de mudanças quando apropriado
- Documentar decisões importantes e seus motivos

## Referências

- README.md na raiz do projeto contém link para esta documentação
- Estrutura inspirada no MkDocs para organização e navegação
